◆ MV_USB_SetTransferSize()

MV_CAMCTRL_API int __stdcall MV_USB_SetTransferSize ( IN void *  handle,
unsigned int  nTransferSize 
)

设置U3V的传输包大小

参数
handle[IN] 设备句柄
nTransferSize[IN] 传输的包大小, Byte，默认为1M，范围：≥0x0800(2K)
返回
成功，返回MV_OK ；失败，返回错误码
备注
增加传输包大小可以适当降低取流时的CPU占用率。但不同的PC和不同USB扩展卡存在不同的兼容性，如果该参数设置过大可能会出现取不到图像的风险。

获取U3V的传输包大小

参数
handle[IN] 设备句柄
pnTransferSize[OUT] 传输的包大小指针, Byte
返回
成功，返回MV_OK ；失败，返回错误码
备注
该接口用于获取当前的U3V传输包大小，默认1M。

◆ MV_USB_SetTransferWays()

MV_CAMCTRL_API int __stdcall MV_USB_SetTransferWays ( IN void *  handle,
unsigned int  nTransferWays 
)

设置U3V的传输通道个数

参数
handle[IN] 设备句柄
nTransferWays[IN] 传输通道个数，范围：1-10
返回
成功，返回MV_OK ；失败，返回错误码
备注
用户可以根据PC的性能、设备出图帧率、图像大小和内存使用率等因素对该参数进行调节。但不同的PC和不同的USB扩展卡存在不同的兼容性。

◆ MV_USB_GetTransferWays()

MV_CAMCTRL_API int __stdcall MV_USB_GetTransferWays ( IN void *  handle,
unsigned int *  pnTransferWays 
)

获取U3V的传输通道个数

参数
handle[IN] 设备句柄
pnTransferWays[OUT] 传输通道个数指针
返回
成功，返回MV_OK ；失败，返回错误码
备注
该接口用于获取当前的U3V异步取流节点个数，2000W设备的MONO8默认为3个，YUV为默认2个，RGB为默认1个，其它情况默认8个节点。
